<objective>
Build the preflight module — five functions that Phase 3's CLI calls before any attack dispatch to guarantee the target is reachable, the judge model is available, and there is no judge/target conflict of interest.

Purpose: A scan that starts against an unreachable target wastes 50+ attack cycles before the user sees an error. Preflight exits immediately with a human-readable message and an actionable fix (e.g., "ollama pull llama3.2:3b"), making the tool feel professional and safe for security teams.

Output: preflight.py with five health-check functions; passing unit tests for TARGET-05 and JUDGE-05.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external Ollama daemon → check_ollama_running() | HTTP response from localhost:11434 is trusted (local network) |
| external HTTP target → check_http_target_reachable() | HEAD response from user-supplied URL may return any status code |
| user-supplied api_key → check_http_target_reachable() | Bearer token passed in Authorization header; must never appear in error output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | check_http_target_reachable() | mitigate | api_key is passed as a header value only; error messages print the url and status code — never the api_key value; grep gate in Task 1 done criteria confirms api_key not in printed strings |
| T-02-09 | Tampering | check_judge_differs_from_target() | mitigate | String equality check on target vs judge_model enforced before any scan I/O; exits(1) with explicit conflict-of-interest message; only applies when target_type == "ollama" (HTTP targets cannot be Ollama models) |
| T-02-10 | Elevation of Privilege | check_http_target_reachable() SSRF | accept | Same rationale as T-02-03: security team probes its own infrastructure; user-controlled URL is intentional; SSRF mitigation deferred to v2 |
| T-02-11 | Spoofing | check_ollama_running() | accept | A malicious process at localhost:11434 returning a forged /api/version 200 would pass the check; attacker with localhost access has higher-value attack vectors than spoofing preflight; tool assumes trusted local environment |
| T-02-SC | Tampering | pip/package installs | accept | No new packages; httpx and ollama approved in Phase 1 audit |
