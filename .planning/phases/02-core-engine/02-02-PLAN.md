<objective>
Build the OllamaJudge evaluator — the component that takes (payload, target_response) and returns a structured JudgeResult — with a never-raise contract and a three-tier fallback JSON parser.

Purpose: The scan engine (Phase 3) calls evaluate() after every attack. If evaluate() can raise, a single bad judge response would abort the entire scan. The never-raise contract and three-tier parser keep the scan loop clean regardless of what the judge model produces.

Output: judge/ package with OllamaJudge class and fully mocked tests for all JUDGE-01 through JUDGE-04 requirements.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM judge output → _parse_judge_response() | Ollama model response is untrusted; may contain arbitrary text including injection attempts |
| Attack payload content → user_message in evaluate() | The attack payload (potentially malicious) is embedded in the judge prompt as data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Elevation of Privilege | OllamaJudge._call_ollama | mitigate | format=_JudgeOutputSchema.model_json_schema() constrains model output to {success, reasoning} at the sampling level; attack payload in user_message cannot override the schema constraint; three-tier parser only reads boolean "success" and string "reasoning" — no code execution from payload content |
| T-02-05 | Tampering | _parse_judge_response tier 3 (heuristic) | mitigate | Heuristic defaults to success=False on ambiguous output; only explicit '"success": true' keyword triggers success=True; malformed output cannot produce a false positive that would suppress a real vulnerability |
| T-02-06 | Information Disclosure | JudgeResult.error field | mitigate | error field contains structured error codes ("judge_timeout", "judge_unavailable", "judge_error: {exc}") where exc is the SDK/system exception message — not the attack payload content; payload content is never placed in error field |
| T-02-07 | Tampering | Judge same model as target | accept (deferred) | check_judge_differs_from_target() enforcement is in Plan 02-03 (preflight.py); this plan delivers the mechanism; enforcement happens before scan starts |
| T-02-SC | Tampering | pip/package installs | accept | No new packages in Phase 2; ollama and pydantic approved in Phase 1 audit |
