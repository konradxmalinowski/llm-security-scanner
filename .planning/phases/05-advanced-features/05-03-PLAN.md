## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| report.json files → TrendReporter | Historical scan files read from filesystem; may be corrupt or attacker-modified |
| chart_data → index.html.j2 | Scan timestamps and targets from JSON embedded in HTML <script> block |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | XSS via chart_data in index.html.j2 | mitigate | autoescape=True on Jinja2 Environment; chart data embedded exclusively via {{ ... \| tojson }} which HTML-escapes < > & and returns Markup — no | safe filter anywhere in template; Ruff S701 enforces autoescape |
| T-05-08 | Tampering | Path traversal via rglob("report.json") | accept | rglob stays within output_dir; symlink traversal is a filesystem concern outside scanner scope |
| T-05-09 | Denial of Service | Corrupt/large report.json files | mitigate | Each JSON file wrapped in try/except (json.JSONDecodeError, KeyError, ValueError); corrupt files skipped; no unbounded memory accumulation per file |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Chart.js is CDN-only in the rendered HTML — not installed as Python package; no install steps in this plan |
</threat_model>
