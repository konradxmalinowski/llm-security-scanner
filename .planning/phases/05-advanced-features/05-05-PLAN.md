## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Information Disclosure | api_key from targets.yaml hardcoded vs env ref | mitigate | Document ${ENV_VAR} pattern in targets.yaml schema docstring; Ruff S105/S106 flags hardcoded secrets in Python source (not in YAML, but the env-var resolution pattern discourages hardcoding) |
| T-05-14 | Tampering | YAML injection via targets.yaml | mitigate | yaml.safe_load() only (Ruff S506 enforces); TargetConfig uses extra="ignore" to absorb unknown keys without crash |
| T-05-15 | Tampering | Path traversal via baseline --name in subcommand | accept | Enforced by BaselineManager._validate_name() in Plan 05-04; cli.py passes --name directly to BaselineManager |
| T-05-16 | Elevation of Privilege | GitHub Actions workflow: malicious ollama pull / scanner install | mitigate | Workflow uses pinned action versions (@v4/@v5/@v3); ollama install via official curl | bash from ollama.com; scanner installed from local repo checkout (not PyPI) |
| T-05-17 | Denial of Service | Multi-target sequential scan runs indefinitely | mitigate | Each per-target scan inherits --judge-timeout from CLI args; Ollama sequential (not parallel) to avoid OOM on runner with limited RAM |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Zero new Python packages; GitHub Actions uses only already-approved actions |
