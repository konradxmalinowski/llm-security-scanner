## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → suppressions.yaml | User-authored file; attacker could craft malicious YAML |
| suppressions.yaml → SuppressionLoader | YAML parsed by PyYAML; content reflected into AttackResult fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | suppressions.yaml YAML parsing | mitigate | Always use yaml.safe_load() — Ruff S506 enforces; never yaml.load() |
| T-05-02 | Tampering | Suppression.attack_id glob injection | accept | fnmatch.fnmatch() is a pure pattern-matching stdlib function with no shell execution; no OS command injection risk |
| T-05-03 | Tampering | AttackResult mutation via apply() | accept | Suppressions only set suppressed=True on findings already in memory; they cannot inject new findings or modify payload/response content |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages in this plan; no install steps |
</threat_model>
